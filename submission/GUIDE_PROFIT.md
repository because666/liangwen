# 收益导向模型 - 完整使用指南

## 📋 问题诊断与解决方案

### 发现的问题
你之前的提交结果显示：
- ✅ **准确率**: 0.50-0.60（看起来不错）
- ❌ **累计收益率**: -76 到 -98（严重亏损）
- ❌ **单次收益率**: -0.0002（每笔交易都在亏钱）

### 根本原因
**"准确率高但收益低"陷阱**：
1. 模型学会了预测多数类（"不变"/class 1），因为这类样本最多
2. 预测"不变" = 不交易 = 没有手续费损失 = 准确率高（因为大多数时候确实不变）
3. 少数方向性预测错误时，亏损抵消了所有潜在利润
4. 结果：**准确率虚高，实际收益为负**

### 解决方案（已实现）

#### 1️⃣ 不对称类别权重 (AsymmetricFocalLoss)
```python
alpha=[2.5, 0.15, 2.5]  # 大幅降低"不变"类权重至0.15
gamma=3.0               # 更强关注困难样本
```
- **效果**：强制模型更多预测涨跌方向，减少"不变"预测

#### 2️⃣ 收益导向损失函数 (ProfitLoss)
```python
weight_matrix = [
    [+2.0, +0.05, -3.0],   # 实际下跌：预测跌=奖励，预测涨=重罚
    [-1.5, +0.05, -1.5],   # 实际不变：预测方向=轻罚
    [-3.0, +0.05, +2.0],   # 实际上涨：预测涨=奖励，预测跌=重罚
]
```
- **效果**：直接优化交易收益，而非分类准确率

#### 3️⃣ 两阶段训练策略
```
Epoch 1-30:   AsymmetricFocalLoss → 学习基本模式
Epoch 31-80:  ProfitLoss → 优化交易收益
```
- **效果**：先学会识别模式，再学习如何盈利

#### 4️⃣ 更保守的出手策略
```python
阈值搜索范围: 0.45-0.85（之前是0.3-0.7）
默认阈值: 0.6（之前是0.5）
```
- **效果**：只在高度确信时交易，提高单次胜率

#### 5️⃣ 收益指标监控
训练时关注的指标从"准确率"变为：
- ✅ **平均单次收益** (avg_profit)
- ✅ **胜率** (win_rate)
- ✅ **出手次数** (trade_count)
- ⚠️ **准确率** (accuracy) 仅作参考

#### 基础训练命令（推荐A10 GPU）
```bash

python train_profit_v2.py \
    --data_dir ../2026train_set/2026train_set \
    --output_dir ./output_profit \
    --epochs 80 \
    --batch_size 256 \

    --use_tta \
```

| `--lr` | 1.5e-4 | 学习率 |
| `--hidden_dim` | 128 | 隐藏层维度 |
| `--dropout` | 0.25 | Dropout率 |
| `--max_files` | 1200 | 使用的最大文件数 |
| `--use_amp` | True | 使用混合精度训练 |
| `--augment` | True | 使用数据增强 |
| `--use_tta` | True | 使用TTA（测试时增强） |
| `--tta_rounds` | 5 | TTA轮数 |

## 🔧 故障排查

### 问题1：显存不足 (OOM)
**症状**：`RuntimeError: CUDA out of memory`

**解决方案**：
```bash
# 减小batch_size
python train_profit_v2.py --batch_size 128

# 或减小hidden_dim
python train_profit_v2.py --hidden_dim 96
```

### 问题2：单次收益仍为负
**症状**：`★ 平均单次收益: -0.000123`

**解决方案**：
```bash
# 1. 增加训练轮数
python train_profit_v2.py --epochs 120

# 2. 调整学习率
python train_profit_v2.py --lr 1e-4

# 3. 增加dropout防止过拟合
python train_profit_v2.py --dropout 0.3
```

### 问题3：出手率过低 (<20%)
**症状**：模型几乎不交易

**解决方案**：
```bash
# 在config.json中手动降低阈值（训练后修改）
# 将thresholds中的值从0.6改为0.5
```

### 问题4：出手率过高 (>60%)
**症状**：模型过于激进，手续费侵蚀利润

**解决方案**：
```bash
# 在config.json中手动提高阈值
# 将thresholds中的值从0.6改为0.7
```

## 📞 常见问题

### Q1: 为什么准确率下降了？
**A**: 这是我们故意的设计！之前的高准确率是虚假的（通过预测"不变"获得）。现在的低准确率是因为我们在做真实的方向性预测。

### Q2: 如何判断模型好坏？
**A**: 只看一个指标：**单次平均收益**。只要这个值 > 0，模型就有正期望。

### Q3: 需要训练多久？
**A**: A10 GPU上大约2-4小时（80 epochs）。建议至少训练60个epoch让两阶段策略都发挥作用。

### Q4: 可以继续用旧的best_model.pt吗？
**A**: 不建议。旧模型是用错误的目标函数训练的，即使准确率高也不会有好的收益。
