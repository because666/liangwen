# 阿里云PAI-DSW环境配置指南

## 问题分析

当前IsaacLab环境的PyTorch版本（2.0.0）与transformers库有冲突，导致训练脚本无法运行。

**错误原因**：
- PyTorch 2.0.0 的 `_pytree` 模块API不兼容
- transformers库尝试调用不存在的 `register_pytree_node` 函数

## 解决方案

### 方法一：使用PAI内置镜像（推荐）

创建DSW实例时选择正确的镜像：

1. 进入PAI控制台 → 创建DSW实例
2. **镜像选择**：
   - 选择 `pytorch:2.1.0-gpu-py310-cu121-ubuntu22.04`
   - 或者 `pytorch:2.2.0-gpu-py310-cu121-ubuntu22.04`
3. 不要使用IsaacLab相关的镜像

### 方法二：在当前实例中创建新环境

在DSW终端执行：

```bash
# 1. 创建新的conda环境
conda create -n deeplob python=3.10 -y

# 2. 激活环境
conda activate deeplob

# 3. 安装PyTorch (CUDA 12.1)
pip install torch==2.1.0 torchvision==0.16.0 torchaudio==2.1.0 --index-url https://download.pytorch.org/whl/cu121

# 4. 安装其他依赖
pip install numpy==1.24.3 pandas==2.0.3 pyarrow==14.0.1 tqdm==4.66.1

# 5. 验证安装
python -c "import torch; print(f'PyTorch: {torch.__version__}'); print(f'CUDA: {torch.cuda.is_available()}'); print(f'GPU: {torch.cuda.get_device_name(0) if torch.cuda.is_available() else \"无\"}')"
```

### 方法三：使用脚本自动配置

```bash
# 上传 setup_env.sh 到DSW
# 然后执行
chmod +x /mnt/workspace/TEMP-FILE-STATION/良文杯/submission/setup_env.sh
bash /mnt/workspace/TEMP-FILE-STATION/良文杯/submission/setup_env.sh
```

## 配置完成后运行训练

```bash
# 激活环境
conda activate deeplob

# 进入目录
cd /mnt/workspace/TEMP-FILE-STATION/良文杯/submission

# 创建数据软链接（如果还没创建）
ln -s /mnt/workspace/TEMP-FILE-STATION/良文杯/2026train_set/2026train_set ./data

# 运行训练
python train_cloud.py \
    --data_dir ./data \
    --output_dir ./output \
    --epochs 100 \
    --batch_size 512 \
    --lr 1e-3 \
    --model_type enhanced
```

## 环境检查清单

配置完成后，运行以下命令检查：

```bash
# 检查Python版本
python --version  # 应该是 3.10.x

# 检查PyTorch
python -c "import torch; print(f'PyTorch: {torch.__version__}')"

# 检查CUDA
python -c "import torch; print(f'CUDA可用: {torch.cuda.is_available()}')"

# 检查GPU
nvidia-smi
```

## 常见问题

### Q: conda命令找不到
```bash
# 使用source激活
source /opt/conda/bin/activate
# 或者
. /opt/conda/etc/profile.d/conda.sh
```

### Q: pip安装慢
```bash
# 使用阿里云镜像
pip install torch==2.1.0 --index-url https://mirrors.aliyun.com/pypi/simple
```

### Q: 显存不足
```bash
# 减小batch_size
python train_cloud.py --batch_size 256  # 或 128
```

## 快速开始步骤

1. **创建DSW实例**（选择正确的镜像）
2. **上传代码和数据**
3. **配置环境**（如果镜像不是最新的PyTorch）
4. **运行训练**

建议重新创建一个DSW实例，选择 `pytorch:2.1.0-gpu-py310-cu121` 镜像，这样可以避免所有环境问题。
